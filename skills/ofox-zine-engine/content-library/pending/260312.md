### 小红书 A 版（洞察向）

**标题**：这个 AI 助手用得越久越聪明，关键还开源

昨晚随手翻 GitHub Trending，看到 NousResearch 的 Hermes Agent 又冲上来了。一天涨了 1200 多 star。

点进去看了半小时，说下我觉得最有意思的地方。

**跟现在大多数 Agent 不一样的点**

现在市面上的 Agent 工具，基本是你给任务，它做完，结束。下次再给类似任务，它从头来过。没有积累。

Hermes Agent 不一样。它每次解决一个复杂问题，会自动把解题过程写成一个技能文档。下次碰到类似的事，直接调这个技能，不重复摸索。

听起来简单对吧？但你仔细想，这其实是「经验」这个概念在 Agent 上的实现。

**几个实际的点**

部署门槛很低。一行命令装完，$5/月的 VPS 就能跑。不绑你的笔记本电脑——你在 Telegram 上发消息给它，它在服务器上干活。

模型随便换。Nous Portal、OpenRouter 上两百多个模型、Kimi、GLM、OpenAI，都行。不锁定单一厂商。

Telegram、Discord、Slack、WhatsApp 都能连。就像你在微信群里 @ 一个同事，只不过这个同事 24 小时在线，而且会自己变强。

**我的判断**

Agent 赛道现在太卷了，每周都有新项目。但大部分是「能用一次」的工具，不是「能用一年」的助手。

Hermes Agent 走的是后面那条路：持久记忆 + 技能积累 + 自我进化。用得越久价值越高。这个方向我觉得是对的。

MIT 协议，NousResearch 团队出品（做 Hermes 系列模型的那帮人），73 个贡献者，不是个人玩具项目。

通过 ofox.ai 接入 OpenRouter 或者其他模型，Hermes Agent 的模型成本可以压得很低。

### 小红书 B 版（专业向）

**标题**：Hermes Agent 技术拆解：一个 $5 VPS 就能跑的自进化 Agent

NousResearch/hermes-agent，5.4k⭐，MIT 协议。GitHub 上连续热了快两周了。

**核心架构**

跟大部分 Agent 框架最大的区别：它不是一个 SDK，是一个跑在服务器上的常驻进程。

```
用户消息（Telegram/Slack/CLI）
  → 网关进程接收
  → 路由到对应技能或通用推理
  → 执行（Local/Docker/SSH/Modal）
  → 记忆写入 + 技能更新
  → 返回结果
```

五种执行后端可选：本地跑、Docker 隔离、SSH 远程、Singularity 容器、Modal 无服务器。安全层面做了 read-only root、capability drop、PID limit、namespace 隔离。

**技能系统**

这是它最核心的设计。每次处理复杂任务后，Agent 自动把解法沉淀为 SKILL.md 文件，遵循 agentskills.io 开放标准。

意味着：
1. 技能可跨实例共享（你的 Agent 学会的，可以给别人用）
2. 社区技能库直接导入（agentskills.io / GitHub / ClawHub）
3. 格式统一，不绑定特定框架

**记忆机制**

两层设计：
- Honcho 辩证用户建模：不是简单存对话记录，是构建用户偏好和工作模式的动态模型
- FTS5 全文会话搜索：所有历史对话可检索，LLM 辅助总结

**模型支持**

不锁厂商。Nous Portal、OpenRouter（200+模型）、z.ai/GLM、Kimi、MiniMax、OpenAI、自定义端点。`hermes model` 一条命令切换，不改代码。

**部署成本**

$5/月 VPS 可以跑。Modal 无服务器模式空闲时接近零成本。也支持 GPU 集群做 RL 训练（Atropos 集成）。

**跟其他 Agent 对比**

| 特性 | Hermes Agent | OpenClaw | Claude Code |
|------|-------------|----------|-------------|
| 运行方式 | 服务器常驻 | 本地网关 | CLI |
| 技能进化 | 自动生成 | 手动配置 | 手动配置 |
| 通信渠道 | 5+ 即时通讯 | 本地集成 | 终端 |
| 模型绑定 | 无 | 无 | Anthropic |
| 部署门槛 | $5 VPS | 本地硬件 | 订阅制 |

**开发者视角**

做 Agent 产品的团队可以关注它的技能标准 agentskills.io。如果这个标准跑通了，Agent 之间共享能力就有了基础设施。这比单个 Agent 好不好用更重要。

通过 ofox.ai 统一接口接入多家模型，可以给 Hermes Agent 最灵活的模型选择空间。

我翻完了整份报告，挑几个最扎眼的数字聊聊。

**ChatGPT：还是老大，但没那么稳了**

9 亿周活跃用户。一年涨了 5 亿。Web 端流量是第二名 Gemini 的 2.7 倍。单看绝对值，没人能打。

但翻到增速那页，画风变了。

换个角度：Claude 的付费用户现在是 ChatGPT 的八分之一，Gemini 是四分之一。基数小增速快，这话没错。但增速差到这个程度，说明用户确实在分流。

**最让我意外的几条**

Claude Code 半年做到 10 亿美元年化收入。开发者工具从来没有这个速度。

Midjourney 从 Top 10 跌到 46。图片生成已经从独立赛道变成 CapCut、Canva 的内置功能了。单独做图片工具，越来越难。

Sora 日活 300 万，下载速度比当年 ChatGPT 还快。视频生成正在接力图片生成。

**人均 AI 采用率：美国排第 20**

前三是新加坡、阿联酋、香港。美国排第 20。

这个数据颠覆了不少人的认知。AI 应用的普及不等于 AI 研发的领先。小市场反而因为数字化程度高，采用更快。

**我的看法**

AI 消费市场正在从「ChatGPT + 一堆小工具」变成「ChatGPT / Claude / Gemini 三足鼎立 + 垂直工具被大工具吞」。
