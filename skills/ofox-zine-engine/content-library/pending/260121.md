### 标题
让 AI 帮你写代码的 5 个黄金法则｜硅谷开发者实战总结

### 正文

用 Cursor、Claude 写代码，总感觉改来改去效率更低？

看了两位硅谷开发者的分享，终于明白问题出在哪了👇

**【事实】**
Jon Kaplan 和 Eric Zakariasson 总结了 5 条 AI 编程心法：

1️⃣ **永远先开 Plan Mode**
先用大白话告诉 AI 你要干嘛，确认计划再动手。省得写完发现方向错了。

2️⃣ **频繁开新对话**
话题一杂，AI 的上下文就乱了。每个任务开个新 chat，干净利落。

3️⃣ **让 AI 做 Code Review**
问它："我这次改动有没有影响到其他功能？"比自己肉眼查靠谱。

4️⃣ **计划同步，执行异步**
规划阶段你得盯着，对齐了就丢给 Cloud Agent 跑，你去干别的。

5️⃣ **建验证环境**
让 Agent 自己测自己的改动，形成自检闭环。

**【背后逻辑】**
这 5 条的核心就一句话：**你是指挥官，不是打字员。**

计划阶段必须你主导，因为方向错了，AI 跑得越快错得越离谱。
执行阶段放手让 AI 干，因为体力活它比你快 10 倍。

**【对你的意义】**
下次用 AI 写代码前，先问自己：
- 我清楚要它做什么吗？（没想清楚就别开写）
- 这轮对话是不是太长了？（超过 10 轮就该换了）
- 我让它自检过吗？（别当人肉测试机）

### 标题
AI Agent 编程进阶：5 个提升 10x 效率的工程实践

### 正文

整理了 @aye_aye_kaplan 和 @ericzakariasson 关于 AI Agent 编程的方法论，适合已经在用 Cursor/Claude/Copilot 的开发者👇

**【概述】**
核心思想：将 AI Agent 视为「需要明确指令的初级工程师」，而非全能助手。

**【5 个技术要点】**

**1. Plan Mode First**
- 在 Cursor 中使用 Plan Mode，先生成实现计划
- 用自然语言 iterate 直到计划合理，再触发代码生成
- 收益：减少无效生成，节省 token 消耗

**2. Context Window 管理**
- 单次对话控制在 10 轮以内
- 切换任务 = 新建 Chat
- 原因：上下文污染导致 Agent 行为漂移

**3. AI-Assisted Code Review**
- Prompt 示例：`scan the changes on my branch and confirm nothing is impacted outside of my feature flag`
- 配合 Bugbot 等工具做回归检测
- 适用场景：feature flag 隔离的增量开发

**4. Sync Plan, Async Implement**
- 计划阶段：同步交互，实时对齐
- 实现阶段：交给 Cloud Agent 异步执行
- 关键：计划对齐的置信度决定异步执行的成功率

**5. Validation Environment**
- 为 Agent 配置自检能力（lint/test/type-check）
- 让 Agent 执行完自己验证 diff
- 效果：减少人工 review 负担，提前拦截低级错误

**【开发者视角】**
这套方法论的本质是**人机协作的边界划分**：
- 高层决策（What/Why）：人类主导
- 低层实现（How）：Agent 执行
- 质量保障：双向验证
