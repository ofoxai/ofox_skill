### 小红书A版（洞察向）

**标题**：「Google官方出手」一行命令，控制整个Workspace

开发者被Google Workspace折腾了多久？

Drive、Sheets、Calendar、Gmail……每个产品一套接口文档，一套认证流程，一套SDK——光调通就要半天。

Google官方终于出手了。

**gws** — 一个覆盖全部Workspace服务的命令行工具，3.6k星，本周登上GitHub Trending。

它最关键的设计：**直接从Google Discovery Service动态生成命令**。

Discovery Service是Google自己维护的接口元数据仓库。gws读这份数据，实时构建CLI命令——每当Google更新Workspace，gws命令自动同步，永不过期。

不用记文档，不用等SDK更新。

```bash
# 列出Drive文件
gws drive files list --params '{"pageSize": 10}'

# 创建表格
gws sheets spreadsheets create --json '{"properties": {"title": "Q1预算"}}'

# 查看接口结构
gws schema drive.files.list
```

它还内置了40+个Agent技能，支持MCP协议——直接插进你的AI工作流，让Agent自主操作你的Google Workspace。

凭证存储用AES-256-GCM加密，支持Service Account、OAuth、无头认证，CI/CD环境直接用。

给AI Agent装上Google Workspace的手——这可能是目前最简洁的方案。

### 小红书A版（洞察向）

**标题**：「阿里达摩院」Agent记忆管理工具今天涨了345颗星

AI Agent忘事是个老大难问题。

对话一长就撑不住了，上下文窗口满了，要么截断历史，要么爆Token费用——这是每个做Agent的人都绕不过的坑。

阿里达摩院旗下AgentScope团队做了一个专门解决这件事的工具：**ReMe**（Remember Me, Refine Me）。

今天在GitHub Trending爆发，单日新增345星，总计1,634星。

它的核心思路非常反直觉：**把记忆存成Markdown文件**。

不是塞进向量数据库的黑盒，而是你能直接打开、直接编辑、直接看懂的文本文件。记忆透明了，问题就好排查了。

```python
from reme.reme_light import ReMeLight

reme = ReMeLight(working_dir=".reme", max_input_length=128000)

# 压缩工具输出（防止上下文爆炸）
messages = await reme.compact_tool_result(messages)

# 压缩历史对话为摘要
summary = await reme.compact_memory(messages=messages)

# 语义搜索记忆
result = await reme.memory_search(query="用户偏好", max_results=5)
```

三种记忆类型：个人记忆（用户偏好）、任务记忆（当前目标）、工具记忆（执行历史）——分开管，按需检索。

检索用向量+BM25混合，比纯向量准。

口号是"Memory as files, files as memory"——记忆是文件，文件就是记忆。

**标题**：ReMe — Agent记忆管理，阿里AgentScope团队出品

**Star**：1,634（今日 +345）| Apache 2.0
**团队**：AgentScope（阿里达摩院）

---

**两套内存系统**

① **ReMeLight（文件型）**：
- 记忆以Markdown格式存储（可读可编辑）
- 对话历史 → 结构化摘要checkpoint
- 工具结果压缩（防止单次输出撑爆上下文）
- 向量 + BM25混合语义检索
- Token感知的会话管理

② **ReMe（向量型）**：
- 个人记忆 / 任务记忆 / 工具记忆三层统一管理
- 自动摘要、增删改查操作

---

**安装与使用**

```bash
# 文件型（轻量）
pip install -e ".[light]"

# 向量型
pip install -U reme-ai
```

```python
reme = ReMeLight(
    working_dir=".reme",
    max_input_length=128000,
    memory_compact_ratio=0.7,
)
await reme.start()

messages = await reme.compact_tool_result(messages)
summary = await reme.compact_memory(messages=messages)
result = await reme.memory_search(query="Python preference", max_results=5)
```

**Python 3.10+ | Apache 2.0**

---

**开发者视角**

关键差异：文件存储比向量数据库更可观测。记忆出问题时，直接打开`.reme/`目录查日志，不用调试黑盒。

`memory_compact_ratio=0.7`意味着上下文用到70%时自动触发压缩——很适合长任务Agent场景。
