### 小红书 B版（专业向）

Anthropic 发布 Claude Code Channels（Research Preview），将编程 Agent 接入即时通讯平台。

**功能架构**

Channel 本质是一个 MCP Server，通过事件推送机制将外部消息注入运行中的 Claude Code 会话。

工作流程：
1. 创建 Telegram/Discord Bot
2. `/plugin install telegram@claude-plugins-official`
3. 配对码验证绑定你的账号
4. `claude --channels plugin:telegram@claude-plugins-official` 启动

消息到达后，Claude 读取事件 → 执行操作（改代码/跑测试/查日志）→ 通过同一 Channel 回复。

**安全设计**

- Sender Allowlist：只有配对验证过的用户能推送消息
- Enterprise 控制：`channelsEnabled` 需管理员显式开启
- 权限继承：Channel 消息触发的操作仍走 Claude Code 权限系统

**vs OpenClaw 架构对比**

| | Claude Code Channels | OpenClaw |
|---|---|---|
| 核心架构 | MCP Plugin → 事件注入 | Gateway 守护进程 + 心跳 |
| 会话模式 | 会话存活时接收 | 常驻后台 |
| 平台数 | 2（Telegram/Discord） | 50+ |
| 编程能力 | Opus 4.6（SWE-bench 80.8%） | 依赖接入模型 |
| 安全记录 | 沙箱 + SOC2 | CVE-2026-25253（CVSS 8.8） |
| 运维依赖 | Bun + claude.ai 登录 | 自部署，需维护 |

**判断**

Claude 不是在做 OpenClaw 的替代品。它是把专业开发者最高频的场景——"远程控制编程会话"——做成了一等公民功能。

50 个平台集成是 OpenClaw 的护城河，但对开发者来说，Telegram + Discord 已经覆盖了 90% 的沟通场景。

当前限制：Research Preview 阶段，仅支持 Anthropic 官方维护的插件白名单。
