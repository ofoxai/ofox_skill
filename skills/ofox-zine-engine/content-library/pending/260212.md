### 小红书 B 版

**标题：智谱 GLM-5：从 355B 到 744B，Agent 工程的新基建**

**概述**

智谱 AI 发布 GLM-5，参数规模从 355B（32B 激活）跃升至 744B（40B 激活），训练数据增至 28.5T tokens。官方定位为"Agentic Engineering"基座模型，专注长链路 Agent 任务执行能力。

这不是简单的参数堆叠，而是架构层面的重构。

**技术要点**

核心升级集中在三个维度：

1. **架构创新**
   - 引入 DeepSeek Sparse Attention，降低部署成本的同时保持长文本性能
   - 新框架 "Slime"：异步强化学习，支持更大规模模型训练
   - 200K 上下文窗口 + 128K 输出长度，真正的长程推理能力

2. **Agent 能力强化**
   - SWE-bench-Verified 达到 77.8（开源 SOTA）
   - Terminal Bench 2.0 得分 56.2，超越 Gemini 3.0 Pro
   - BrowseComp、MCP-Atlas、τ²-Bench 三个 Agent 基准测试第一

3. **开发者友好**
   - 原生支持 Function Calling 和工具调用
   - 内置 Model Contextualization Protocol (MCP)
   - 支持流式输出、结构化 JSON、上下文缓存

**开发者视角**

从实际使用角度，GLM-5 的价值体现在两个层面：

**对 Agent 框架开发者**：
- 长链路任务分解能力显著提升，不再需要过度拆解复杂任务
- MCP 协议原生支持，降低多模型切换成本
- Context Caching 机制优化对话成本，适合长交互场景

**对应用开发者**：
- Agentic Coding 能力达到接近 Claude Opus 4.5 的水平
- 办公自动化场景（多步骤工作流）逻辑连贯性强
- 数据提取和质检场景精准度高，适合生产环境落地

真正的差异点在于：GLM-5 在复杂推理 + 工具调用的组合任务上表现突出，这正是当前 Agent 框架的核心瓶颈。

**专业问题**

📌 从 SWE-bench 和 Terminal Bench 的分数看，GLM-5 的代码生成能力确实到了新高度。

但在生产环境中，你会直接用 GLM-5 替换现有的 Agent 底座吗？还是会采用多模型组合策略？

如果考虑成本 vs 性能的平衡，GLM-5 的最佳适用场景是什么？
