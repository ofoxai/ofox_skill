## 260317-01

### 主题
OpenViking：字节开源 Agent 上下文数据库

### 事实清单
- 仓库：volcengine/OpenViking（Apache-2.0）
- Star：14,684（3/15 登顶 GitHub Trending #1，24h +1,610）
- 语言：Python 82.7% + C++ + Rust
- 核心：用文件系统范式（viking:// 协议）统一管理 Agent 的记忆/资源/技能
- L0 摘要 ~100 tokens → L1 概览 ~2k tokens → L2 完整数据，按需加载
- 三目录：Resources（项目文档）/ User（个人偏好）/ Agent（技能/任务记忆）
- 自演化：会话结束后异步分析执行结果+用户反馈，自动更新记忆
- 性能：OpenClaw 集成任务完成率 +43%（51.23% vs 35.65%），Token 减少 91%
- 集成：OpenClaw / OpenCode / Claude 记忆插件 / Doubao / OpenAI / LiteLLM
- 安装：pip install openviking

### 小红书A版（洞察向）

你的 AI Agent 是不是每次都像失忆一样？

字节跳动刚开源了 OpenViking，一个专门给 Agent 用的"上下文数据库"。

它用文件系统的思路管理 Agent 的所有上下文：
viking://resources/ 放项目文档
viking://user/ 放你的偏好习惯
viking://agent/ 放技能和任务记忆

最妙的是三层按需加载——
L0：一句话摘要（100 tokens），快速判断要不要深入
L1：核心信息（2k tokens），规划阶段用
L2：完整数据，真正需要时才读

实测效果：接入 OpenClaw 后任务完成率提升 43%，Token 消耗直降 91%。

这意味着 Agent 不再是每次从零开始，而是像人一样——先回忆大概，需要细节再翻档案。

而且它会"自演化"：每次会话结束自动分析执行结果，更新记忆库。越用越聪明，不用你手动维护。

### 小红书B版（专业向）

字节开源 OpenViking：Agent 记忆终于有了"文件系统"

volcengine/OpenViking | ⭐ 14.6k | Apache-2.0

一句话：用 viking:// 虚拟文件系统统一管理 Agent 上下文（记忆/资源/技能），三层分级加载。

核心设计：

1️⃣ 文件系统范式
所有上下文统一为 viking://{scope}/{path} 路径
Resources/User/Agent 三个根目录各司其职
像管文件一样管记忆，可浏览、可检索、可版本化

2️⃣ L0/L1/L2 分层加载
L0 摘要（~100 tokens）→ 快速索引
L1 概览（~2k tokens）→ 规划决策
L2 完整数据 → 按需深读
避免一次性灌入全量上下文吃掉 token 预算

3️⃣ 自演化机制
会话结束后异步分析执行结果和用户反馈
自动更新 User/Agent 目录，无需手动维护

性能数据（官方 README）：
- OpenClaw 集成：任务完成率 51.23% vs 35.65%（+43%）
- Token 消耗：2.1M vs 24.6M（降 91%）
- 对比 LanceDB：准确率高 15-17%，Token 成本降 92-96%

安装：pip install openviking
集成：OpenClaw / OpenCode / Claude 插件 / Doubao / OpenAI / Ollama

### 小红书A版（洞察向）

AI 写代码最大的问题不是"不够聪明"，是"看不懂你的项目"。

GitNexus 换了个思路：不用向量搜索，而是把整个代码库建成知识图谱。

每个函数调用、依赖关系、继承链、执行流程，全部映射成图结构。AI 查代码时用 Cypher 图查询，沿着关系链精准定位。

实际使用：
npx gitnexus analyze → 自动扫描项目生成图谱
Claude Code / Cursor 通过 MCP 直接查询
"这个函数改了会影响哪些模块？" → 沿调用链秒答

支持 TS/JS/Python/Java/Go/Rust/C/C++/C# 九种语言。

这等于给 AI 装了一双"结构化的眼睛"——不再是在文件里盲目搜索关键词，而是沿着代码的逻辑脉络去理解。

## 260317-03

### 主题
Hindsight：让 Agent 从经验中学习的记忆系统

### 事实清单
- 仓库：vectorize-io/hindsight（MIT）
- Star：4,416
- 语言：Python
- 核心：Agent 记忆系统，重点是学习而非仅记住
- 论文：arxiv 2512.12818 "Hindsight is 20/20"
- LongMemEval：91.4%（基线 39%），经 Virginia Tech + Washington Post 独立验证
- LoCoMo：89.61%（前最佳 75.78%）
- 三类记忆：World（世界事实）/ Experiences（代理经验）/ Mental Models（心智模型）
- 三大操作：Retain（存储）/ Recall（检索）/ Reflect（反思）
- 召回：语义+关键词+图谱+时序 四路并行 → 倒数排名融合 → cross-encoder 重排
- 核心技术：TEMPR（时间实体记忆+上下文感知召回）+ CARA（代理反思机制）
- 集成：Claude Code / Cursor / MCP / Pydantic AI / Ollama / OpenAI / Gemini
- 安装：pip install hindsight-client -U 或 Docker
- 公司：Vectorize, Inc.（Boulder, Colorado）

### 小红书A版（洞察向）

大多数 Agent 记忆系统在做的事：帮 AI 记住聊天记录。

Hindsight 做的事不一样：让 AI 从经验中学习。

区别在哪？

普通记忆 = "上次聊了什么"
Hindsight = "上次做的决策对不对，下次怎么做更好"

它把记忆分成三层：
World — 世界事实（某项目用 Python 3.12）
Experiences — 执行经验（上次部署踩了什么坑）
Mental Models — 心智模型（这个用户偏好什么风格）

检索时四路并行跑：语义、关键词、图谱、时序，最后 cross-encoder 重排选最相关的。

LongMemEval 基准测试拿了 91.4%（基线才 39%），Virginia Tech 和 Washington Post 独立验证过。

最实用的是：它支持 Claude Code、Cursor、MCP，装上就能让你的编程 Agent 越用越懂你的项目。

pip install hindsight-client -U

MIT 开源，可商用。
