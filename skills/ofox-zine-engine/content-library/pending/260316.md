### 小红书 A版（洞察向）

**标题**：GLM-5-Turbo 周日突袭，智谱在抢什么？

智谱周日发了一个新模型 GLM-5-Turbo。选在周末发布，说明这事儿挺急。

**不是新模型，是 GLM-5 的"Agent特调版"。**

GLM-5 发布一个月后，智谱把同一个 744B 参数底座（40B 激活）拿出来，专门针对 Agent 场景做了四项优化：

1. **工具调用不掉链子** — Agent 最怕调工具时断掉，这是核心痛点
2. **复杂指令拆解** — 多层嵌套的长指令，拆得更准
3. **定时持续执行** — 跑一整晚的任务不崩
4. **长链路吞吐稳定** — 几十步的 Agent 流程也能撑住

200K 上下文、128K 最大输出，定价 $1.2/$4 每百万 token。

**为什么值得关注？**

OpenClaw 生态里的 Skills 使用比例从 26% 飙到 45%。这意味着越来越多人在用 Agent 技能而不只是聊天。GLM-5-Turbo 就是为这个场景量身定制的。

上线第一天就收到 38 万次请求，说明开发者确实在等这样的模型。

**洞察**：模型竞争正在从"谁更聪明"转向"谁更能干活"。不是跑分高就赢了，而是 Agent 工作流里谁更稳定、更持久。通过 ofox.ai 可以直连 GLM-5-Turbo，体验国产 Agent 模型的实际效果。

### 小红书 B版（专业向）

**标题**：GLM-5-Turbo 技术拆解：Agent 长链路优化 + 200K 上下文

智谱 2026-03-15 发布 GLM-5-Turbo，定位为 GLM-5 的 Agent 场景专项优化版本。

**模型规格**

| 参数 | 数值 |
|------|------|
| 基础模型 | GLM-5（744B 总参/40B 激活，MoE） |
| 上下文窗口 | 202,752 tokens（~200K） |
| 最大输出 | 131,072 tokens（~128K） |
| 输入价格 | $1.2 / M tokens |
| 输出价格 | $4 / M tokens |

**四项 Agent 优化**

| 维度 | 说明 |
|------|------|
| 工具调用 | 增强外部工具与技能调用的稳定性 |
| 指令理解 | 复杂/多层/长链路指令拆解能力提升 |
| 时间维度 | 定时触发、持续执行场景专项优化 |
| 高吞吐 | 长链路执行效率与响应稳定性提升 |

**支持特性**

思考模式（`<think>` 标签）、Function Call、MCP 工具集成、结构化输出（JSON）、上下文缓存、流式输出

**评测表现**

ZClawBench 基准中相比 GLM-5 提升显著，OpenClaw 场景多项任务领先主流模型。生态数据：Skills 使用比例从 26% → 45%。

**上线数据（首日）**

| 指标 | 数值 |
|------|------|
| 请求数 | 381,987 |
| Prompt tokens | 16.5B |
| 推理 tokens | 41.7M |

**调用方式**

- OpenRouter：`z-ai/glm-5-turbo`
- 直接调用：`open.bigmodel.cn/api/paas/v4/chat/completions`
- SDK：zai-sdk / zhipuai（Python），Maven（Java）
- 通过 ofox.ai 可统一接入，无需单独注册智谱账号

**与同类对比**

GLM-5-Turbo 的差异化不在通用跑分，而在 Agent 工作流的"持久力"。对于需要长时间运行、多步工具调用的 Agent 应用，这是目前国产模型中针对性最强的选择。
