### 主题
Karpathy autoresearch — AI 自动跑 ML 实验

### 事实清单
- 作者：Andrej Karpathy
- GitHub：karpathy/autoresearch，10.6k stars
- 协议：MIT
- 语言：Python 83.4%
- 依赖：PyTorch + uv 包管理，无其他外部依赖
- GPU：单张 NVIDIA GPU（H100 测试）
- 核心机制：AI Agent 修改 train.py → 训练 5 分钟 → 检查是否改进 → 保留或丢弃 → 重复
- 实验速度：~12 次/小时，一晚 ~100 次
- 核心指标：val_bpb（validation bits per byte，越低越好）
- 核心文件：train.py（Agent 改）、program.md（人改）、prepare.py（不动）
- 安装：`uv sync` → `uv run prepare.py`（~2分钟）→ `uv run train.py`（~5分钟）
- 设计哲学："One GPU, one file, one metric"
- 来源：https://github.com/karpathy/autoresearch

### 小红书 A 版（洞察向）

**标题**：你睡了一觉，AI 帮你跑了 100 个实验

Karpathy 刚开源了一个叫 autoresearch 的项目，思路非常简单：

给 AI 一个小型 LLM 训练环境，让它自己改代码、跑实验、看结果，循环往复。你睡一觉起来，它已经迭代了 100 轮。

**它是怎么工作的？**

AI Agent 读取 program.md 里的研究方向 → 修改 train.py → 训练 5 分钟 → 检查 val_bpb 是否下降 → 好就保留，差就回滚 → 继续下一轮。

一小时跑 12 个实验，一晚上 100 个。全程只需要一张 GPU。

**为什么这件事很重要？**

过去做 ML 研究，90% 的时间花在等训练、调参数、记录结果。autoresearch 把这个循环自动化了——不是替代研究者的直觉，而是替代重复的试错过程。

人决定研究方向（写 program.md），AI 负责暴力搜索（改 train.py）。分工清晰。

**对你意味着什么？**

如果你在做模型训练相关的工作，这可能是目前最值得关注的范式变化：研究的瓶颈从"算力不够"变成了"研究方向对不对"。

一张 GPU + 一个晚上，就能验证几十个假设。门槛极低，MIT 协议。

### 小红书 B 版（专业向）

**标题**：autoresearch：单 GPU 自动化 ML 研究框架

Karpathy 开源了 autoresearch（10.6k⭐），一个让 AI Agent 自主进行 ML 训练实验的框架。

**核心架构**

项目只有三个关键文件：
- `train.py`：完整的 GPT 模型 + 优化器 + 训练循环（Agent 可修改）
- `program.md`：研究指令（人类编写，指导 Agent 方向）
- `prepare.py`：数据准备（固定不动）

**工作流程**

```
Agent 读取 program.md
  → 修改 train.py
  → 训练 5min（固定时间预算）
  → 评估 val_bpb
  → 改进则保留，否则回滚
  → 循环（~12次/小时）
```

**技术要点**

- 评估指标：val_bpb（bits per byte），与词表大小无关
- 依赖极简：PyTorch + uv，无分布式训练、无复杂配置
- GPU 要求：单张即可（H100 测试）
- 已有社区 fork 支持 macOS 和 Windows

**设计哲学**

"One GPU, one file, one metric" — 刻意保持极简，让 Agent 的搜索空间可控。人定义方向，Agent 暴力搜索参数空间。

**开发者视角**

这不是又一个 AutoML 框架。它的独特之处在于：Agent 直接修改源码，而不是调超参。这意味着它能探索架构层面的改进，而不只是 learning rate 搜索。

MIT 协议，安装三步完成：`uv sync` → `uv run prepare.py` → `uv run train.py`

## 260309-03

### 主题
OpenClaw 小白门槛真相 — 你以为下载就能用？

### 事实清单
- OpenClaw（龙虾）：开源 AI Agent 系统，非简单聊天客户端
- 源码安装需要 Node.js 20+ 环境，普通人连"终端"是什么都不知道
- QQ 已上线一键部署龙虾机器人，1 分钟可用（解决 90% 小白安装问题）
- 腾讯云/阿里云几乎同时推出一键部署服务
- 大模型接口：豆包/千问/DeepSeek 有免费额度，但需注册开发者账号获取 key
- 搜索能力默认不支持，需自行配置 Brave Search（免费 2000次/月）
- 安全风险（工信部已发高危预警）：
  - CVE-2026-25253：一键远程代码执行漏洞，点链接就被入侵
  - 3万+ OpenClaw 实例暴露在公网，无认证保护
  - ClawHub 技能市场约 20% 是恶意技能（800+个）
  - Moltbook 泄露 150 万 API token + 3.5 万邮箱
  - API key 明文存储，第三方部署可截获你的 key
- 来源：The Register / DigitalOcean / Bitsight / 工信部预警 / 知乎

### 小红书 A 版（洞察向）

**标题**：OpenClaw 真的是小白能用的吗？你有没有这些疑问和问题

全网都在"养龙虾"，我也跟风折腾了一阵。

结论是：如果你不是搞技术的，这玩意儿真没有博主说的那么轻松。我把自己踩过的坑和身边朋友问过的蠢问题都列出来了，你看看中了几个。

**"终端"是啥？在哪打开？**
教程第一步就是"打开终端输入命令"。问题是——终端在哪？很多人连电脑上有这个东西都不知道。Windows 叫命令提示符或 PowerShell，Mac 叫终端。光找到它就要搜半天，更别说往里面敲命令了。

**为什么不能像微信一样下载就用？**
这是最常见的困惑。豆包、千问、Kimi，下载 App 注册就能聊。但龙虾不是一个 App，它更像一套"组装电脑"——你得自己装系统、接显示器、连网络。这就是为什么有人 10 分钟搞定，有人折腾一整天。

**"接口"到底是什么？免费模型为什么还要花钱？**
这个问题我被问了不下十次。简单说：你平时用的豆包、千问是"餐厅堂食"（免费但排队限量），接口是"食材批发"（按斤买但你得自己做饭）。去各家官网注册开发者账号就能拿到 key，DeepSeek、千问、豆包都有免费额度，入门不花钱。

**配好了，但它怎么这么慢？还不能语音？**
如果你接的是国外模型，网络延迟会很高。换成国内模型（DeepSeek、千问、豆包）基本秒回。语音输入确实不支持，用手机自带的语音转文字打到输入框里，曲线救国。

**想让它帮我搜东西，搜不了？**
龙虾默认没有联网能力。想搜索要额外配 Brave Search，又是注册账号又是填 key。Brave 每月免费 2000 次够日常用，但配置过程又会卡住一批人。

**连微信/QQ好麻烦，每个都不一样**
想把龙虾接到微信、QQ、飞书、钉钉——每个渠道的机器人创建流程完全不同，权限申请也是一堆坑。建议别同时搞，先挑一个最常用的跑通。QQ 已经有官方一键部署了，最省事。

**最吓人的：安全风险你可能完全不知道**
这才是我最想说的。工信部已经发了高危预警，龙虾目前有严重的安全问题：
- 你填进去的模型 key 是明文存储的，用别人搭的服务等于把 key 交出去
- ClawHub 技能市场大约 20% 是恶意技能，安装了可能泄露你的数据
- 全网有 3 万多个龙虾实例裸奔在公网上，没有任何认证保护
- 今年 1 月曝出的 CVE-2026-25253 漏洞，点一个链接就能被远程控制

很多小白用户根本不知道这些风险，兴冲冲装完就把自己的 key 和数据暴露了。

**那还能不能用？**
能用，但得看你怎么用：
- 怕麻烦 → 用 QQ 官方龙虾机器人或云服务商一键部署
- 自己搭 → 一定要设认证密码，别裸奔；别装来路不明的技能
- key 安全 → 只用自己部署的实例，不在别人的龙虾服务里填自己的 key

说白了，龙虾是给动手能力强的人准备的 DIY 套件。想要"下载即用"的体验，不如直接选封装好的成熟工具——用着省心，数据也安全。

### 小红书 B 版（专业向）

**标题**：OpenClaw 上手避坑指南：门槛 + 安全 + 解法

OpenClaw 爆火后涌入大量非技术用户，但真实上手门槛远超预期。以下是按阶段拆解的痛点和对应方案。

**第一关：安装（劝退 90% 的人）**
- 源码需要 Node.js 20+、Git 环境，普通人连终端都不会打开
- 解法：QQ 已上线官方一键龙虾机器人；腾讯云/阿里云有一键部署镜像

**第二关：认知（接口是什么？key 是什么？）**
- "豆包免费为什么还要买接口"是最高频问题
- 解法：接口 = 程序调用通道，和网页版是两回事。DeepSeek/千问/豆包官网注册开发者账号即可，都有免费额度

**第三关：配置（每个渠道都不一样）**
- 微信/QQ/飞书/钉钉的机器人创建流程各异，权限体系也不同
- 解法：先跑通一个渠道（推荐 QQ 官方方案），再逐步扩展

**第四关：体验（慢、不能语音、不能搜索）**
- 国外模型延迟高 → 换国内模型秒回
- 语音输入不支持 → 系统语音转文字代替
- 搜索需要 Brave Search → 免费 2000次/月，注册填 key 即可

**第五关：安全（工信部高危预警）**
- CVE-2026-25253：一键 RCE 漏洞
- 3万+ 实例无认证暴露公网
- ClawHub 约 20% 恶意技能（800+个）
- Moltbook 泄露 150万 token + 3.5万邮箱

- 解法：只用自部署实例 + 设认证 + 不装来路不明技能 + 不在他人服务填 key

核心结论：龙虾是 DIY 工具，自由度高但风险也高。普通用户建议选官方渠道部署或成熟的封装服务，别为了省钱把数据安全搭进去。
