### 主题
Anthropic内部曝光：Claude Code数百个Skills的分类体系与最佳实践

### 事实清单
- 来源：Thariq (@trq212) X 长文，Claude Code 团队核心成员
- 作者身份：Claude Code @anthropicai，前 YC W20，MIT Media Lab
- 系列：Lessons from Building Claude Code 第三篇（前两篇：Seeing like an Agent / Prompt Caching Is Everything）
- 发布日期：2026-03-17
- 数据：315万阅读、10,279点赞、29,045收藏、1,317转发、236评论
- 核心结论：Anthropic 内部使用数百个 Skills，分类后发现聚成几个反复出现的类别
- Skills 6大分类：
  1. Library & API Reference — 教 Claude 正确使用某个库/CLI/SDK，含参考代码片段+常见坑列表
  2. Product Verification — 描述如何测试/验证代码正确性，搭配 Playwright/tmux 等外部工具
  3. Code Quality & Review — 组织内代码质量强制+代码审查，可含确定性脚本确保最大健壮性
  4. Scaffolding — 处理自然语言需求到代码模板（new-framework-workflow/new-migration/create-app）
  5. Data Fetching & Analysis — 连接数据和监控栈，含凭证管理和常见工作流说明
  6. Infrastructure Operations — 日常运维标准流程，让工程师执行关键操作时遵循最佳实践
- 最佳实践要点：
  - Gotchas 章节是 Skill 中信号最高的内容，应从 Claude 常见失败点积累并持续更新
  - Verification Skills 极其有用，值得工程师花一整周专门打磨
  - Skill 是文件夹（含脚本/资产/数据），不只是 Markdown 文件
  - 最好的 Skill 清晰归入单一分类，混杂多个分类的 Skill 往往让人困惑
- 原文链接：https://x.com/trq212/status/2033949937936085378

### 小红书A版（洞察向）

Anthropic 内部用了几百个 Claude Code Skills，他们发现了什么规律？

Claude Code 核心开发者 Thariq 刚发了一篇长文，首次公开 Anthropic 内部的 Skill 分类体系。

这是"Lessons from Building Claude Code"系列第三篇，315万人看了，2.9万人收藏。

他们把所有 Skill 分成了 6 类：

📚 知识型 — 教 AI 正确用某个库，附代码片段和"常见坑"清单
✅ 验证型 — 搭配 Playwright/tmux 自动验证代码是否正确
🔍 质量型 — 强制代码审查标准，用确定性脚本兜底
🏗️ 脚手架型 — 一句话需求生成完整项目模板
📊 数据型 — 连接监控栈，含凭证管理工作流
⚙️ 运维型 — 标准化日常操作，防止关键步骤出错

最有价值的发现：
一个 Skill 里信号最高的部分是 Gotchas（常见坑）章节。它应该从 AI 反复犯的错误里积累，并持续更新。

他还说，Verification Skills 值得工程师花一整周专门打磨——因为验证 AI 输出的正确性，比让 AI 写代码更难。

### 小红书B版（专业向）

Claude Code 核心开发者公开内部 Skill 分类体系

来源：Thariq @trq212 | 315万阅读 | 2.9万收藏

一句话：Anthropic 从数百个生产 Skill 中提炼出 6 大分类 + 核心最佳实践。

6 大分类：

1️⃣ Library & API Reference
教 Claude 正确调用某库/CLI/SDK
关键：附参考代码 + Gotchas 常见坑列表

2️⃣ Product Verification
搭配 Playwright/tmux 等工具验证输出
Thariq 原话：值得工程师花一整周专门做好

3️⃣ Code Quality & Review
组织级代码质量强制执行
可嵌入确定性脚本做硬性检查

4️⃣ Scaffolding
从自然语言需求到项目模板
例：new-framework-workflow / create-app

5️⃣ Data Fetching & Analysis
连接数据/监控栈
含凭证管理 + 常见查询工作流

6️⃣ Infrastructure Operations
日常运维标准化
让关键操作不再凭记忆

核心洞察：
- 最好的 Skill 只归一个分类，跨类的往往让人困惑
- Skill 不只是 Markdown，是文件夹（含脚本/资产/数据）
- Gotchas 章节信号最高，应从 AI 常见失败点持续积累
- 这是该系列第三篇（前两篇：Seeing like an Agent / Prompt Caching）

这系列文章就是 Claude Code 的设计哲学白皮书。
