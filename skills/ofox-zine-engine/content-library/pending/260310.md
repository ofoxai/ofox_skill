## 260310-01

### 主题
CollectivIQ — 多模型共识平台，用 10 个 AI 投票替代单模型赌博

### 事实清单
- 定位：世界首个 AI 共识平台（AI Consensus Platform）
- 发布：2026-03-04 从隐身模式出来（TechCrunch Exclusive）
- 总部：波士顿，孵化于 Buyers Edge Platform
- 融资：$47M Series A，A16Z 领投，Lightspeed + YC 参投
- 创始团队：Mehta（前 Google AI 安全团队负责人）+ Chen（前 DeepMind 模型集成研究总监）
- 支持模型：ChatGPT、Claude、Gemini、Grok + 最多 10 个其他模型
- 核心机制：并行查询多模型 → 识别一致/矛盾点 → 融合共识答案
- 隐私：Sanitizer Engine™ 自动剥离 PII → 零数据留存 → TLS 1.3 + AES-256
- 合规目标：SOC 2 / GDPR / ISO 27001 / HIPAA（2026）
- 定价：Business tier $999/月，约 $0.08/query（比单独调用省 77%）
- 企业功能：角色权限 + 审计日志 + 无状态 API 架构 + 自定义知识库
- 来源：https://techcrunch.com/2026/03/04/ / https://www.collectiviq.ai/

### 小红书 B 版（专业向）

**标题**：CollectivIQ：同时问 10 个 AI，用共识替代幻觉

CollectivIQ 3月4日出隐身，A16Z 领投 $47M，定位"AI 共识引擎"。

**核心机制**

传统路径：选一个模型 → 祈祷它不瞎编。
CollectivIQ 路径：同时查询 ChatGPT/Claude/Gemini/Grok 等最多 10 个模型 → 交叉比对 → 标注共识区和矛盾区 → 输出融合答案。

本质是把"选最强模型"变成"让多个模型互相纠错"。

**技术架构**

- Sanitizer Engine™：查询发出前自动剥离 PII（姓名/邮箱/手机号）
- 零数据留存：全程 ephemeral 处理，不日志不训练
- 无状态 API：企业级接入，不是消费级聊天界面
- 加密：TLS 1.3 传输 + AES-256 静态加密

**定价逻辑**

Business tier $999/月，单次查询约 $0.08。看着贵，但拆开算：同时调 10 个模型的接口成本远高于此，CollectivIQ 批量议价后反而省 77%。

**创始团队**

Mehta（前 Google AI 安全 / 事实性 grounding 团队）+ Chen（前 DeepMind 模型集成研究）。两个人的背景精准对应产品的两个核心：安全过滤 + 多模型融合。

**开发者视角**

多模型路由不是新概念（OpenRouter 早就做了），但 CollectivIQ 的差异化在"共识层"——不是选最快/最便宜的模型，而是让多个模型对同一个问题交叉验证。对高风险场景（医疗/法律/金融）的企业客户，这比任何单模型的置信度都高。

通过 ofox.ai 的统一接口，你也能实现多模型并行调用，自己搭建轻量级的共识验证流程。

## 260310-02

### 主题
MiroFish — 群体智能预测引擎，1000 个 AI 人格模拟平行世界

### 事实清单
- GitHub：666ghj/MiroFish，11.7k stars / 1.2k forks
- 今日增长：+2,294 stars
- 协议：AGPL-3.0
- 语言：Python 57.8% + Vue 41.1%
- 背景：盛大集团战略投资
- 核心引擎：OASIS（Open Agent Social Interaction Simulations，来自 CAMEL-AI）
- 依赖：OpenAI 兼容 LLM API（推荐阿里通义千问 Qwen-Plus）+ Zep Cloud 记忆管理
- 五步流程：图谱构建（GraphRAG）→ 环境搭建（实体提取+人格生成）→ 模拟运行（双平台并行+动态记忆更新）→ 报告生成（ReportAgent+工具集成）→ 深度交互（对话模拟实体）
- 输入：种子材料（新闻/政策/报告/小说）+ 自然语言需求
- 输出：预测报告 + 可交互的数字世界
- 应用场景：舆情预测、金融推演、政策影响评估、创意叙事
- 示例：武汉大学舆情模拟、《红楼梦》结局预测
- 安装：`npm run setup:all` → `npm run dev`
- 运行环境：Node.js 18+ / Python 3.11-3.12 / Docker 支持
- 来源：https://github.com/666ghj/MiroFish

### 小红书 B 版（专业向）

**标题**：MiroFish：用 1000 个 AI 人格搭建平行世界的预测引擎

666ghj/MiroFish（11.7k⭐，今日 +2.3k），基于多 Agent 群体智能的预测框架，盛大集团战投。

**核心思路**

不是"问一个 AI 预测未来"，而是：构建一个平行数字世界 → 注入上千个有独立人格和记忆的 Agent → 让它们自由互动演化 → 你从"上帝视角"注入变量 → 观察群体行为走向。

本质是把预测问题转化为社会模拟问题。

**五步架构**

```
种子信息（新闻/政策/财报）
  → GraphRAG 图谱构建
  → 实体提取 + Agent 人格生成
  → OASIS 引擎双平台并行模拟（动态记忆更新）
  → ReportAgent 生成预测报告
  → 深度交互（和模拟实体对话）
```

**技术栈**

- 模拟引擎：OASIS（CAMEL-AI 的开源社会模拟框架）
- 记忆管理：Zep Cloud（Agent 长期记忆持久化）
- LLM：OpenAI 兼容接口（推荐通义千问 Qwen-Plus）
- 前端：Vue 构建的可视化交互界面
- 部署：Docker 或源码，Node.js 18+ / Python 3.11-3.12

**应用场景**

- 舆情预测：给定事件种子，模拟公众反应演化
- 金融推演：注入政策变量，观察市场参与者行为
- 政策评估：模拟政策落地后的社会反馈链条
- 创意叙事：输入小说背景，让 Agent 自行演绎结局（《红楼梦》案例）

**开发者视角**

MiroFish 的野心不小——用社会模拟替代传统预测模型。优势是可解释性强（你能看到每个 Agent 的决策链），劣势是对 LLM 调用量巨大（1000 个 Agent 交互 = 海量 token）。

如果你想用低成本跑这类高调用量的多 Agent 模拟，通过 ofox.ai 接入通义千问等国内模型是个高性价比选择。AGPL-3.0 协议，商用需注意。

## 260310-03

### 主题
Claude Code Review — 多智能体自动审查 PR，16% → 54% 的审查覆盖率跃升

### 事实清单
- 发布：2026-03-09，研究预览（Research Preview）
- 可用性：Team 和 Enterprise 计划
- 架构：多智能体并行审查 → 交叉验证去误报 → 按严重性排序输出
- 输出形式：单条概览评论 + 具体问题行内标注
- 审查覆盖率：16% → 54% PR 获得实质性审查意见
- 大 PR（1000+ 行）：84% 发现问题，平均 7.5 个
- 小 PR（<50 行）：31% 发现问题，平均 0.5 个
- 误报率：<1% 被标记为不正确
- 审查时间：约 20 分钟
- 定价：$15-25/PR，按 token 计费
- 管理功能：月度组织消费上限 + 仓库级开关 + 分析仪表盘
- 背景数据：Anthropic 内部工程师代码产出同比增长 200%
- 关键限制：不会 approve PR，最终决策仍需人工
- 检测类型：身份验证逻辑缺陷、类型不匹配、被修改代码中的潜在问题
- 启用方式：管理员在 Claude Code 设置中启用 → 安装 GitHub App → 选择仓库
- 来源：https://claude.com/blog/code-review / https://techcrunch.com/2026/03/09/

### 小红书 A 版（洞察向）

**标题**：AI 写的代码越来越多，谁来审？Claude 的答案是派一队 Agent

Anthropic 昨天发了 Code Review，不是又一个 lint 工具，是直接往 PR 里派一队 AI Agent 做代码审查。

**背景**

Anthropic 自己内部的数据很说明问题：工程师代码产出同比涨了 200%。代码越写越多，但 reviewer 没有变多。之前只有 16% 的 PR 能收到实质性审查意见——剩下的要么没人看，要么看了点个 approve 走人。

上了 Code Review 之后，这个数字变成 54%。

**怎么做到的**

不是一个大模型从头到尾扫一遍。是多个专业 Agent 并行工作：
- 有的专门找身份验证逻辑漏洞
- 有的专门看类型不匹配
- 有的检查你改动的代码附近有没有潜在隐患
- 最后一轮交叉验证，把误报过滤掉

结果：误报率不到 1%。1000 行以上的大 PR，84% 都能被揪出问题，平均 7.5 个。

**关键细节**

审一次大概 20 分钟，$15-25 一个 PR。不便宜，但算一下：一个高级工程师花 40 分钟做 review 的人力成本是多少？而且 AI 不会因为赶 deadline 就草草 approve。

有个设计选择很有意思：**它不会 approve PR**。审查意见给你，approve 的权力留给人。这不是技术限制，是产品设计——Anthropic 明确把最终决策权留给开发者。

**对你意味着什么**

AI 编程工具的发展路线越来越清晰：先帮你写代码（Copilot/Claude Code），再帮你审代码（Code Review），接下来是帮你测代码。从"辅助创作"到"质量保障"，AI 在开发流程里的角色从助手变成了同事。

对团队来说，真正的价值不是省 reviewer 的时间，是**让那些以前没人看的 PR 也被认真审了**。
