Anthropic 3月12日发布了 Claude 的交互可视化能力（Interactive Visuals），从产品设计到技术实现都值得关注。

**功能概述：**
Claude 现在能在对话中内联生成交互式图表、图示和动态可视化。不需要用户写任何代码，模型自主判断何时需要视觉辅助。

1. 对话 UI 的复杂度大幅提升——纯文本渲染不够了
2. 这对 Claude 的工具调用能力提出了更高要求

**和竞品对比：**
ChatGPT 的 Canvas 是侧边面板+代码执行，Claude 选择了内联+无代码的路径。两种产品哲学——ChatGPT 偏向"给你工具"，Claude 偏向"替你理解"。

### 小红书A版（洞察向）

**标题：5个AI在一个"论坛"里吵架，吵出来的是舆情报告**

正文：

GitHub上有个38000⭐的项目叫 BettaFish（微舆），做了一件很有意思的事：

**让5个AI Agent在一个论坛里互相辩论，最后吵出一份舆情分析报告。**

不是一个AI看完所有数据给你写总结。而是5个各有专长的Agent——有的负责搜新闻，有的负责看短视频，有的负责挖内部数据——先各自分析，然后坐下来"开会讨论"。

还有一个主持人模型负责协调辩论，确保观点不跑偏。

为什么这个设计有意思？

因为**单个AI的分析一定有盲区**。它可能搜索能力强但不懂视频内容，可能擅长中文舆情但漏掉了海外讨论。

5个Agent各有视角，辩论过程本身就是一种自我纠错。这比"一个AI做所有事"要可靠得多。

更有意思的是它和另一个项目 MiroFish 是上下游关系——BettaFish 负责"发生了什么"，MiroFish 负责"接下来会怎样"。从数据采集到趋势预测，形成闭环。

这种"Agent辩论"的设计模式，可能会成为下一代AI应用的标配。

### 小红书B版（专业向）

**标题：BettaFish 38k⭐ 多Agent舆情系统架构拆解**

正文：

BettaFish（微舆）是一个基于多Agent协作的舆情分析系统，38.4k⭐，架构设计有不少值得学习的地方。

**5个Agent分工：**

| Agent | 职责 | 特点 |
|-------|------|------|
| QueryEngine | 国内外新闻搜索 | 广度覆盖 |
| MediaEngine | 多模态内容理解 | 短视频+图片 |
| InsightEngine | 私有数据挖掘 | 内部数据库 |
| ReportEngine | 报告迭代生成 | 多轮优化 |
| ForumEngine | 辩论协调 | 主持人模型 |

**核心设计——Agent论坛机制：**
不是简单的Agent串行调用，而是让多个Agent在"论坛"结构中辩论协作。ForumEngine 作为主持人，引导讨论方向，综合各方观点。

**技术特点：**
- 30+ 社交媒体24/7 AI爬虫集群监控
- 多模态处理：短视频、图片、搜索引擎数据卡片
- 公域+私域数据融合分析
- 纯Python框架，模块化设计，支持单引擎独立测试（Streamlit）
- Docker + PDF导出

**与 MiroFish 的关系：**
BettaFish（采集/分析）→ MiroFish（预测），构成完整的"感知-预测"流水线。

**开发者参考价值：**
- Agent辩论机制是多Agent协作的一种新范式
- 5 Agent分工 + 主持人协调的模式，适合迁移到其他复杂分析场景
- 如果你要做类似系统，通过 ofox.ai 接口可以为不同Agent分配不同模型（比如搜索用快模型，分析用强模型）
