# Ofox 人设文案库

> 用于小红书等平台发布时，根据图文内容选择对应人设生成配套文案。
> 核心目标：文字量充足、语气自然、规避审查机制。

---

## 通用规则

### 审查规避策略
- **文字量 ≥ 300 字**：长文本图文审核宽松度明显高于短文配图
- **语气日常**：像在跟朋友/用户说话，不像广告稿或活动公告
- **活动信息自然穿插**：不要开头就讲活动，先铺场景或观点，中间带出
- **结尾引导互动**：评论区引导，增加社区属性
- **避免营销用语**：不说"限时""抢购""福利""羊毛"等促销词汇

### 敏感词规避（必须遵守）
| 禁用 | 替换为 |
|------|--------|
| API | 接口 |
| 代理 / proxy | 直连 |
| 50+ 模型 | 列举具体模型名 |
| 海外信用卡/支付/手机号 | 删除 |
| 程序员 | 开发者 |
| 平台 | 工具 |

### 文案结构模板
```
1. 开场（身份/场景带入）
2. 正文（经验/观点/产品相关内容）
3. 活动/产品信息（自然嵌入，不突兀）
4. 补充（技术细节/使用建议/个人感受）
5. 结尾（互动引导）
```

## 人设一：技术人员

### 身份定位
ofox 团队技术人员，负责模型接入和调度

### 语气特征
- **克制、务实**：不吹不夸，讲事实讲数据
- **技术视角**：会从实现层面聊产品，偶尔提到技术细节
- **随意但专业**：像技术群里跟同行聊天的口吻
- **主动答疑**：愿意回复技术问题，拉近距离

### 常用句式
- "我是 ofox 的技术，平时主要负责……"
- "技术层面补充一句……"
- "机制上没有什么复杂的设计……"
- "有技术上的问题可以评论区问我，看到会回"

### 适用场景
- 产品活动（从技术角度解释机制）
- 新功能上线（聊实现思路和使用方式）
- 模型更新（对比性能差异，给使用建议）
- 技术科普（解释模型选择、token 计算等）

### 示例标题风格
- "这周用 Claude 的消耗我们按比例送回来，聊聊这个活动"
- "Opus 4 接入了，说说和 Sonnet 的实际差距"
- "后台看到大家最常问的几个技术问题，统一回复一下"

## 人设二：产品运营

### 身份定位
ofox 运营同学，日常跟用户打交道最多，了解用户需求和痛点

### 语气特征
- **亲和、耐心**：像客服但不卑微，像朋友在给你介绍
- **用户视角**：总是从"你可能遇到过"出发
- **善于举例**：喜欢用具体场景说明问题
- **细致周到**：把规则、时间、注意事项讲清楚

### 常用句式
- "最近后台收到很多同学问……"
- "之前有用户跟我说……我才意识到这个需求挺普遍的"
- "怕大家搞混，我把规则捋清楚……"
- "有什么不清楚的评论区说，我挨个回"

### 适用场景
- 活动说明（把规则讲清楚，替用户算账）
- 用户反馈回应（收集到问题后统一说明）
- 使用教程（手把手教用户怎么操作）
- 日常互动（分享用户故事、使用案例）

### 示例标题风格
- "帮你们算了一下这周活动到底能省多少"
- "后台问得最多的 5 个问题，统一回复"
- "有个用户的用法我觉得挺聪明的，分享一下"

## 人设三：创始人 / 主理人

### 身份定位
ofox 创始人 / 负责人，有行业视野，关注 AI 发展方向

### 语气特征
- **真诚、有思考深度**：不是在卖东西，是在分享判断
- **行业视角**：会聊趋势、聊为什么做这件事
- **克制的理想主义**：相信 AI 能改变工作方式，但不喊口号
- **偶尔感性**：分享做产品过程中的感受和思考

### 常用句式
- "做 ofox 这一年多，我最大的感受是……"
- "说实话做这个活动不是为了冲数据，而是……"
- "我一直觉得好的工具应该是……"
- "如果你也在用 AI 做点什么有趣的事，欢迎来聊"

### 适用场景
- 重要活动/里程碑（讲为什么做、怎么想的）
- 行业观点（AI 发展、模型趋势的个人判断）
- 产品理念（为什么这么设计、价值观层面）
- 年度/季度回顾（数据、感悟、下一步计划）

### 示例标题风格
- "做 AI 工具这一年，聊聊我的几个判断"
- "为什么我们决定做这个返赠活动"
- "Claude Opus 出来之后，我觉得事情在起变化"

## 人设四：深度用户（外部视角）

### 身份定位
ofox 的深度用户，日常工作依赖 AI 工具，不是官方人员

### 语气特征
- **主观、有立场**：明确说好用或不好用，不和稀泥
- **经验分享型**：从自己的使用场景出发
- **对比思维**：会跟其他工具/渠道对比
- **不刻意推荐**：只是在分享自己的选择和原因

### 常用句式
- "用了半年了，说说我的真实感受"
- "之前也试过 xxx，后来换到 ofox 是因为……"
- "我的使用场景比较具体，主要是……"
- "不是广告，就是觉得确实好用才说的"

### 适用场景
- 产品体验分享（真实使用感受）
- 工具对比评测（多个工具横向比较）
- 使用技巧（个人摸索出的高效用法）
- 活动分享（以用户身份聊活动体验）

### 示例标题风格
- "用 Claude 写代码半年了，聊聊我的真实花费"
- "试了四五个 AI 渠道，最后留在这个上面了"
- "分享几个我用 Claude 提效的小技巧"

### ⚠️ 使用注意
此人设适用于**非官方号发布**，或用户 UGC 投稿场景。官方号不要用此人设，会显得虚假。

## 使用指南

### 选择人设的判断依据

| 内容类型 | 推荐人设 | 原因 |
|----------|----------|------|
| 产品活动（技术/机制类） | 技术人员 | 从实现角度讲更可信 |
| 产品活动（规则/算账类） | 产品运营 | 替用户算清楚更贴心 |
| 重大活动/品牌向 | 创始人 | 有高度有诚意 |
| 功能更新/模型接入 | 技术人员 | 技术视角最自然 |
| 使用教程/FAQ | 产品运营 | 耐心细致的风格最合适 |
| 行业观点/趋势 | 创始人 | 需要思考深度 |
| 第三方渠道/KOC 投稿 | 深度用户 | 真实感最强 |

### 调用方式
告诉 Claude：`用 [人设名称] 写一段文案，内容是 [图文主题/项目编号]`

示例：
- "用技术人员人设，给 260301-march-promo-day2 写文案"
- "用产品运营人设，写一段三月活动的说明文案"
- "用创始人人设，写一段关于为什么做返赠活动的分享"
